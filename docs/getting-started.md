# Getting started

This page covers running titan-tyr locally and running its test suite.

For the full design, see [`DESIGN.md`](../DESIGN.md). For the endpoint
reference, see [`api.md`](./api.md).

## Prerequisites

- Python 3.11+
- Docker (for Postgres in dev and for the test container)

## Set up a virtualenv

```sh
python3 -m venv .venv
source .venv/bin/activate
pip install -e ".[test]"
```

## Run a Postgres for development

```sh
docker run --rm -d --name titan-tyr-pg \
  -p 5432:5432 \
  -e POSTGRES_USER=titan \
  -e POSTGRES_PASSWORD=titan \
  -e POSTGRES_DB=titan_tyr \
  postgres:16-alpine
```

## Apply migrations

```sh
DATABASE_URL='postgresql+asyncpg://titan:titan@localhost:5432/titan_tyr' \
  alembic upgrade head
```

Alembic is wired to `src/models.Base.metadata`, so future migrations
can be authored with `alembic revision --autogenerate -m "<message>"`.
Per [`DESIGN.md`](../DESIGN.md#migrations), migrations are expected to
run as a step *before* the API starts (entrypoint command, init
container, or compose dependency) — never inside FastAPI startup.

## Run the API

```sh
DATABASE_URL='postgresql+asyncpg://titan:titan@localhost:5432/titan_tyr' \
  uvicorn src.main:app --reload --port 8000
```

The OpenAPI docs are auto-generated by FastAPI:
- Swagger UI: <http://localhost:8000/docs>
- ReDoc:      <http://localhost:8000/redoc>
- Raw schema: <http://localhost:8000/openapi.json>

Every endpoint requires `Authorization: Bearer sysmlv2`.

## Smoke-test it

```sh
curl -s -H 'Authorization: Bearer sysmlv2' \
  http://localhost:8000/templates/software
```

Then register a piece of software:

```sh
curl -s -H 'Authorization: Bearer sysmlv2' \
     -H 'Content-Type: application/json' \
     -d '{"name":"payments-service","repo_uri":"https://example.com/repo","markdown":"# payments-service"}' \
     http://localhost:8000/software
```

## Run the tests

```sh
pytest
```

The test suite uses
[testcontainers-python](https://testcontainers-python.readthedocs.io/)
to spin up a throwaway Postgres in Docker. Each test gets a fresh
schema (`drop_all` + `create_all`) — fast and reliable.

If you already have a Postgres available and don't want the container
overhead, point at it with:

```sh
TEST_DATABASE_URL='postgresql+asyncpg://titan:titan@localhost:5432/titan_tyr_test' pytest
```

Coverage is reported by `pytest-cov` and configured in
`pyproject.toml`. The current suite covers 95% of `src/`; the
remaining lines are defensive paths (template-not-on-disk,
contract-with-no-active-version, the engine factories that the test
fixtures override).

## Layout

```
src/
  main.py           FastAPI app + router wiring
  config.py         pydantic-settings
  db.py             Async engine + session + MetaData naming convention
  auth.py           Bearer-password dependency (placeholder)
  models.py         SQLAlchemy ORM
  schemas.py        Pydantic request/response models
  versioning.py     Semver parse/compare/format
  routers/          One module per resource
templates/          Static markdown served by /templates/*
alembic/            Migrations
tests/              Pytest + testcontainers
docs/               You are here
```
